# BUGS.md — PilotOS

> Tracker des bugs et issues connues. Mise à jour à chaque sprint.

## Format

```
## [BUG-NNN] Titre court
- **Statut** : new | confirmed | in_progress | resolved
- **Priorité** : critical | high | normal | low
- **Résolu en** : vX.X (si résolu)
- **Description** : Ce qui se passe vs ce qui devrait se passer
- **Reproduce** : Étapes pour reproduire
- **Note** : Contexte technique
```

---

## Bugs connus (V0 — Initialisation)

Aucun bug connu à ce stade — projet en initialisation.

---

## Règles

1. Tout bug trouvé en test → entrée dans ce fichier avant fix
2. Priorité `critical` → fix dans la journée
3. Priorité `high` → fix dans le sprint en cours
4. Un bug résolu inclut toujours la version de résolution
5. Les doublons sont mergés (vote++ sur l'issue principale)
